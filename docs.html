Introduction

Welcome to the platform documentation. This guide will walk you through everything you need to know to get started, from initial setup to advanced configuration. Whether you're building a new project from scratch or integrating into an existing system, you'll find comprehensive guidance here.

Overview

The platform provides a unified API layer for managing data, authentication, and real-time events across your applications. Built with performance and developer experience in mind, it handles the infrastructure complexity so you can focus on building features your users care about.

javascript 
// Initialize the client
import { createClient } from '@windfield/sdk';

const client = createClient({
  apiKey: process.env.WINDFIELD_API_KEY,
  project: 'my-project',
});

// Fetch data with type safety
const { data, error } = await client
  .from('users')
  .select('id, name, email')
  .limit(10);

Key Features

  • Type-safe SDK — Full TypeScript support with auto-generated types from your schema, eliminating runtime type errors.
  • Real-time subscriptions — Listen for changes in your data with WebSocket-powered subscriptions that scale automatically.
  • Built-in authentication — Row-level security, JWT tokens, and OAuth providers configured out of the box.
  • Edge-first deployment — Run your functions at the edge for sub-50ms response times globally.

Note: This documentation covers version 2.x of the SDK. If you're using version 1.x, refer to the legacy documentation for migration guidance and API differences.
